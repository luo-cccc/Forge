# 当前阶段任务：Phase 2 (Harness 状态机与流式对话)
本阶段的核心目标是：在 Rust 后端建立 Harness 状态机，接入 LLM API，并通过 Tauri Event 将大模型的回复**流式（Streaming）**推送到前端。
请严格按照以下步骤执行，每完成一步请与我确认：

## Step 1: Rust 后端依赖配置
- 在 `src-tauri/Cargo.toml` 中添加必要的依赖，用于发起异步 HTTP 请求和处理 JSON。
- 必须添加：`reqwest` (开启 stream 和 json feature), `tokio`, `serde`, `serde_json`, `dotenvy` (用于读取本地 .env 文件中的 API Key)。

## Step 2: 前端基础双栏布局 (Mock UI)
- 修改前端页面，用 Tailwind 建立一个经典的“左右分屏”布局：
  - 左侧 (占据 2/3 宽度)：一个全屏的大 `textarea`，作为临时的小说编辑器占位。
  - 右侧 (占据 1/3 宽度)：Agent 交互区。包含一个消息展示列表（目前用简单的 div 即可）和一个输入框。

## Step 3: Rust 端 Harness 状态机与 LLM 接入
- 在 Rust 中定义一个 Harness State Enum，至少包含 `Idle`（空闲）, `Thinking`（思考中/请求中）, `Streaming`（流式返回中）。
- 编写一个 Tauri Command (如 `ask_agent`)。该命令接收前端传来的字符串，读取 `.env` 中的 `OPENAI_API_KEY`（或兼容模型的 Key）。
- **核心难点**：使用 `reqwest` 向 OpenAI API 发起流式请求 (stream: true)。
- 在接收数据流的循环中，使用 Tauri 的 `app_handle.emit_all` (v1) 或 `app.emit` (v2) 将每个 chunk（数据块）作为一个自定义事件（如 `agent-stream-chunk`）发送给前端。发送完毕后发送一个 `agent-stream-end` 事件。

## Step 4: 前端流式事件监听
- 在 React 端使用 Tauri 的 `listen` API 监听 `agent-stream-chunk` 事件。
- 每次收到 chunk，将其追加到右侧交互区的当前消息气泡中，实现打字机效果。
- 监听 `agent-stream-end` 事件，结束本次对话状态。
