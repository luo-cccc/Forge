# 当前阶段任务：Phase 11 (Hermes 引擎原生 Rust 化：自我进化与三层记忆，https://github.com/NousResearch/hermes-agent.git)
本阶段的核心目标是：不引入 Python 环境，直接在 Rust 后端重构 Hermes 的“技能萃取（Skill Extraction）”和“用户画像漂移（User Drift）”机制。
请严格按照以下步骤执行，每完成一步请与我确认：

## Step 1: 引入 SQLite 构建持久化语义层 (FTS5)
- **目标**：虽然我们有了处理模糊相似度的 Vector DB，但仍需要精准的结构化记忆和全文检索。
- **任务**：在 Rust 中引入 `rusqlite` crate，并确保开启 `bundled` 和 FTS5 特性。
- 初始化 `hermes_memory.db`，建立三张核心表：
  1. `session_history`: 存储最近的对话轨迹，用于短期意图追踪。
  2. `user_drift_profile`: 存储用户偏好的动态画像（如：偏好短句、喜欢暗黑风格）。
  3. `agent_skills`: 存储 Agent 从历史交互中总结出的“技能法则”。

## Step 2: 技能萃取引擎 (Skill Extraction Loop)
- **核心逻辑**：模仿 Hermes 的后台反思机制。当用户连续对某几段文字进行了精修替换（触发了 `<ACTION_REPLACE>`），或者在消息气泡中明确纠正了 Agent 的文风后，触发一次后台提取。
- **Rust 后端实现**：
  1. 新增一个 `tokio::spawn` 的后台异步任务 `extract_skill_from_interaction`。
  2. 将最近的 N 轮对话和用户的修改记录打包，静默发送给 LLM（或者较小、较便宜的模型）。
  3. Prompt 设计：“作为反思引擎，分析用户刚才的修改记录。提取出 1-2 条可复用的写作规则或用户偏好，并以 JSON 格式输出。”
  4. 将提取出的规则（例如 "Skill: 角色林墨在对话时从不使用感叹号，喜欢反问"）写入 `agent_skills` 表，或保存为 `.skills/` 目录下的 Markdown 文件。

## Step 3: 动态上下文组装网络 (Context Injection Matrix)
- 重构 Harness 状态机中的 Prompt 组装环节。在向 LLM 发起请求前，需要进行“记忆熔合”。
- **组装顺序**：
  1. 基础 System Prompt（角色设定）。
  2. **[新增]** 查询 `user_drift_profile`，注入用户画像：“当前用户偏好：[偏好设定]”。
  3. **[新增]** 根据用户的当前 Query，在 `agent_skills` 中进行轻量级关键词匹配，注入相关技能：“你已掌握的相关技能：[技能列表]”。
  4. Phase 10 的 RAG 向量检索片段。
  5. 当前编辑器的上下文。

## Step 4: UI 层的“顿悟”反馈 (Epiphany Bubbles)
- 当后端的 `extract_skill_from_interaction` 成功萃取并保存了一个新技能时，通过 Tauri Event `agent-epiphany` 推送到前端。
- 在右侧的 AgentPanel 对话流中，插入一个特殊样式的系统消息气泡（例如带有发光边框或特殊动画的小卡片）：
  - 💡 **Agent Learned a New Skill:** *"已掌握：在描写战斗场景时，优先使用短句和动词叠加。"*
  - 气泡上提供一个 `[x]` 或 `[Edit]` 按钮，允许用户驳回或修改这个被 Agent 总结出来的规则。

# 开发纪律（严格遵守）
1. **纯净依赖**：坚决不引入任何 Python 或外部运行时引擎，所有数据库操作通过 `rusqlite`，所有文本处理通过 Rust 标准库和现有的 LLM 请求通道。
2. **非阻塞设计**：技能萃取必须完全异步，绝不能阻塞用户正常的打字和当前的对话流。
3. **小步提交**：跑通“SQLite 数据库初始化与技能表建立”后 commit；跑通“后台萃取并在气泡中显示顿悟提示”后再 commit。
把这个阶段做完，你的工具就不再是一个冰冷的编辑器了。每次用户修改它的错误，它都能发挥真正的“记住”和“学会”，甚至在消息气泡里主动展示它的成长。这对于独立开发者来说，是打造超高用户留存率的终极杀器。
