# 核心架构原则 (The Harness Architecture)
本项目采用类似于“中间件网关”的 Harness 架构：
1. 前端是“哑终端 (Dumb Terminal)”：只负责渲染界面（消息气泡 UI、编辑器）和捕获用户输入。前端不直接调用大模型。
2. 后端是“胖中枢 (Fat Controller)”：Rust 进程作为 Harness 核心，负责管理上下文状态、接收前端事件、调度大模型请求，并将结果流式下发给前端。
3. 双端通信：前后端必须使用 Tauri 原生的 Event 机制进行异步/流式通信。
