# 当前阶段任务：Phase 6 (行内气泡 Cmd+K 与幽灵预览)
本阶段的核心目标是：在左侧编辑器中引入行内交互机制。用户可以通过快捷键在光标处唤醒输入气泡，AI 生成的内容会以高亮预览的形式直接插入文本中，等待用户“接受”或“拒绝”。
请严格按照以下步骤执行，每完成一步请与我确认：

## Step 1: 注册行内输入气泡 (Inline Bubble)
- 依托 Tiptap 编辑器，利用其 `FloatingMenu` 或结合 `tippy.js` / React 状态，开发一个轻量级的输入组件 `InlineCommandBubble.tsx`。
- 绑定快捷键 `Cmd+K` (Mac) 或 `Ctrl+K` (Windows)：按下后，在当前光标的精确坐标处，弹出一个包含输入框的小气泡。
- 气泡 UI 需要极其纯净，只包含一个 Input，输入如“把这段改得更阴森一点”，按 Enter 提交。

## Step 2: 预览样式的 Tiptap Extension (Ghost Text)
- 开发或配置一个 Tiptap Mark 扩展（例如命名为 `AIPreviewMark`），用于渲染带有特定背景色（如淡绿色 `bg-green-100 text-green-800`）的文本。
- 目标：让 AI 流式注入的文本在视觉上与用户的原文严格区分开，表明这是“待确认的草稿”。

## Step 3: 行内请求与流式注入 (Inline Streaming)
- 当用户在行内气泡按下 Enter，前端调用 `invoke("ask_agent")`。
- 修改前端的流拦截器：如果是由 `Cmd+K` 触发的请求，解析出的 `<ACTION_INSERT>` 或 `<ACTION_REPLACE>` 文本，在写入编辑器时，**必须带上 Step 2 定义的 `AIPreviewMark` 标记**。
- 流传输期间，行内气泡可以切换状态为“⚙️ 思考中...”，并提供一个“停止”按钮。

## Step 4: 接受与拒绝机制 (Accept / Reject)
- 流结束后（收到 `agent-stream-end`），在被高亮的 AI 文本末尾（或行内气泡内）提供两个选项：
  - `Accept (Tab)`: 去除绿色的 `AIPreviewMark`，让这段文字变成普通正文，永久保留。
  - `Reject (Esc)`: 触发 Tiptap 的 undo 或直接删除带有该 Mark 的文本区间，恢复原状。

# 开发纪律（严格遵守）
1. **状态隔离**：必须清楚区分“右侧面板聊天”和“编辑器内 Cmd+K”这两种触发源。行内生成的流不应该出现在右侧的聊天历史中，它是一次性、即看即用的。
2. **焦点管理**：弹出行内气泡时，需要管理好编辑器的焦点，确保 AI 注入时依然能找准原始光标位置。
3. **小步提交**：跑通“Cmd+K 唤醒 -> 输入指令 -> 原地生成绿色文本 -> 按 Tab 接受”的流程后，提醒我进行 Git Commit。
